Introducing, isymtope

One template language for client and server

Design your app once and only once.

Redux-ified

Specify your stores and action right in the template, we will generate the Javascript for you, automatically.

API's included.

Define your API's in our template language or import definitions from grpc.io, protobuf, swagger and more. We will generate Redux for each API call, both REST and RPC.

You can also define APIs for your database bindings using diesel.rs.

Make free calls!

Calls to your API's are free when rendering server-side (when the implementation is in the same process). Calls are always treated as asynchronous and exposed to your app as Redux actions.

You write the exact same code to call your services.

No (server-side) Javascript

This framework is pure Rust and can even by compile your entire site into a native binary taking advantage of Rust's zero-cost abstractions. You don't need Node.js or any Javascript framework running on your server.

Remember, isymtope is a work-in-progess, not all of the above features work yet.

Demo 1

use html;

store {
    let counter = 0;

    counter {
        action increment => value + 1;
        action decrement => value - 1;
    }
}

component counter {
    h4 { "Counter: " { counter } }
    p (class="actions") {
        a (href="#decrement") || { dispatch decrement; } { "Decrement" }
        { " | " }
        a (href="#increment") || { dispatch increment; } { "Increment" }
    }
}

counter (x="counter") {}

(Of course, the syntax is still subject to change.)

Getting the code

Currently you will need to clone this repository to get the source, it hasn't been submitted to crates.io yet.

git clone https://github.com/tmzt/isymtope
cd isymtope

cargo build
cargo test

Currently the examples are run as cargo tests.

Running the examples

If you want to build the examples yourself, you can run:

cargo build
cargo test

You will need to install Rust if you haven't already.

curl https://sh.rustup.rs -sSf | sh

Then you will find the html pages with the demos in the output folder of your checked out repository.
